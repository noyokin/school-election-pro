# Школьные выборы PRO 4.0.8

Полноценная система школьного электронного голосования на PHP 8.1+ и SQLite. Работает в XAMPP без MySQL.

Подробный список возможностей: [FEATURES.md](FEATURES.md).
Инструкция обновления старой версии: [UPGRADE.md](UPGRADE.md).
Инструкция для Windows и XAMPP: [README-WINDOWS.md](README-WINDOWS.md).

## Требования XAMPP

Проверьте расширения:

```bash
/Applications/XAMPP/xamppfiles/bin/php -m | grep -E "pdo_sqlite|sqlite3|mbstring|zip|dom|fileinfo"
```

Обязательны `pdo_sqlite`, `mbstring`, `fileinfo`. Для импорта и экспорта XLSX также нужны `zip` и `dom`; без них доступен CSV.

## Новая установка

1. Распакуйте папку в:

```text
/Applications/XAMPP/xamppfiles/htdocs/school-election-pro
```

2. Настройте права:

```bash
cd /Applications/XAMPP/xamppfiles/htdocs/school-election-pro
sudo chown -R daemon:daemon data uploads backups exports
sudo chmod -R 775 data uploads backups exports
```

3. Запустите Apache в XAMPP Manager.
4. Откройте:

```text
http://localhost/school-election-pro/setup.php
```

5. После установки войдите:

```text
http://localhost/school-election-pro/admin/login.php
```

## Установка на Windows

Для Windows 10/11 и XAMPP подготовлена отдельная инструкция:

[README-WINDOWS.md](README-WINDOWS.md)

Стандартный путь проекта:

```text
C:\xampp\htdocs\school-election-pro
```

Основная база:

```text
C:\xampp\htdocs\school-election-pro\data\election.sqlite
```

MySQL запускать не требуется.

## Первичная настройка

1. Создайте или отредактируйте кампанию в разделе **Выборы**.
2. Импортируйте учеников в разделе **Ученики**.
3. Добавьте кандидатов.
4. В **Настройках** задайте базовый URL для QR-кодов.
5. Распечатайте карточки доступа.
6. Перед открытием проверьте отчёты и создайте резервную копию.
7. Откройте голосование.

## Шаблон таблицы учеников

Используйте `templates/students-template.xlsx` или CSV. Столбцы:

- Код ученика;
- ФИО;
- Класс;
- Пароль.

Перед импортом система показывает предпросмотр и ошибки. Код и пароль лучше хранить в Excel/Numbers как текст, чтобы не исчезали начальные нули.

В версии 4.0.7 большие таблицы обрабатываются пакетами по 20 строк. После подтверждения отображается процент выполнения, количество добавленных, обновлённых и пропущенных записей. Не требуется увеличивать `max_execution_time` для обработки всего файла одним запросом. Список учеников выводится постранично по 25, 50 или 100 записей.

## Сохранение протокола в PDF

Откройте **Отчёты → Протокол / сохранить PDF**, нажмите печать и в системном диалоге macOS выберите **Save as PDF / Сохранить как PDF**. Это не требует сторонней PDF-библиотеки или шрифтов.


## Массовое удаление

Главный администратор может выполнить безопасное массовое удаление:

- **Ученики → Удалить всех доступных учеников** — удаляет все учётные записи без истории участия; связанные с открытыми или завершёнными выборами записи сохраняются;
- **Кандидаты → Удалить всех кандидатов** — удаляет всех кандидатов выбранной кампании, только если за них ещё нет голосов.

Перед каждой такой операцией система автоматически создаёт резервную SQLite-копию в папке `backups`.

## Резервные копии

Файлы создаются в `backups`. Администратор может скачать их из настроек. Перед восстановлением система автоматически сохраняет текущую базу.

Основная база:

```text
data/election.sqlite
```

## Права XAMPP macOS

При ошибке `unable to open database file`:

```bash
cd /Applications/XAMPP/xamppfiles/htdocs/school-election-pro
sudo mkdir -p data uploads/candidates backups
sudo chown -R daemon:daemon data uploads backups exports
sudo chmod -R 775 data uploads backups exports
```

Проверка:

```text
http://localhost/school-election-pro/diagnostics.php
```

После диагностики удалите или переименуйте `diagnostics.php`, если сайт доступен из внешней сети.

## Безопасность

- Используйте HTTPS при публикации в интернете.
- Не публикуйте папки `data` и `backups` без защиты Apache.
- Используйте отдельные роли администраторов.
- Не редактируйте кандидатов после открытия без крайней необходимости.
- Перед каждым голосованием создавайте резервную копию.
- QR-код не содержит пароль; он только открывает форму с заполненным кодом ученика.


## Аварийное восстановление отсутствующих таблиц

При ошибке `no such table: admins` или `no such table: students` установите патч 4.0.5 и откройте:

```text
http://localhost/school-election-pro/repair.php
```

На странице задайте аварийный логин и новый пароль администратора, затем введите `ВОССТАНОВИТЬ`. Перед изменениями автоматически создаётся копия текущей базы. Подробности: [PATCH-4.0.5.md](PATCH-4.0.5.md).

## Изменения 4.0.8

Версия 4.0.8 ускоряет большие базы учеников и пакетный импорт. Массовое удаление теперь может удалить всех учеников, включая участников завершённых выборов, при этом анонимные голоса и историческая статистика сохраняются в снимках кампаний. Подробности: `PATCH-4.0.8.md`.
